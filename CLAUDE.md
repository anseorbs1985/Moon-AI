# Moon-AI 프로젝트 — 클로드 공통 지침

## 지침 관리 규칙 (클로드 스스로 지킬 것)

- 사용자가 클로드의 행동 방식에 대한 **새 지침**을 내리면, 이 파일(CLAUDE.md)에 추가하고
  **자동으로 GitHub에 push** 한다 (따로 물어보지 않아도 됨). — 단, 아래 push 권한 규칙 적용.
- `git pull`을 받으면 이 파일의 최신 지침을 읽고 **그대로 따른다** — 어느 컴퓨터에서든 동일하게.

## push 권한 (메인 컴퓨터 구분) — 절대 규칙

- **메인 컴퓨터**: launcher 폴더의 `local_config.json`에 `"is_main": true` 가 있는 컴퓨터.
- **메인이 아닌 컴퓨터에서는 사용자가 명시적으로 "올려줘/push"라고 명령하기 전에는
  절대 GitHub에 push하지 않는다** (지침 자동 push 규칙도 메인에서만 적용).
  수정이 생기면 로컬 커밋까지만 해두고, push가 필요하면 사용자에게 알린다.
- **메인 컴퓨터에서는**: 코드 변경 후 배포할 때 **공유 좌표(coords.json·island_coords.json)에
  미푸시 변경이 있으면 함께 커밋·push한다** — 좌표를 로컬에만 두면 다른 컴퓨터가 받아갈 수 없다.

## 자동 업데이트

- 메인런처가 **5분마다 GitHub을 확인**해서 새 커밋이 있으면 자동으로 업데이트를 실행한다
  (작업 실행 중이거나 사용자가 2분 내 입력 중이면 미루고, 한가해지면 실행).
  메인에서 push만 하면 다른 컴퓨터들은 손대지 않아도 자동으로 받아간다.

## 좌표 동기화(병합) 규칙 — 업데이트 시

- 업데이트(git pull)로 coords.json·island_coords.json이 바뀌면 **통째로 덮어쓰지 않고 3-way 병합**한다:
  **이 컴퓨터에서 수정한 슬롯/값은 그대로 유지**하고, **수정하지 않은 것만 원격(GitHub) 값으로** 가져온다.
  (lineagem_update.py의 _merge3 — 슬롯 단위 비교, 업데이트 창에 "로컬 수정 N곳 유지" 표시)

## 사용자 답변 축약어 (모든 컴퓨터 공통)

**응 = ㅇㅇ = dmd = dd** → 전부 "응(yes)" 이라는 뜻이다.
(dmd = '응'을 영문 자판으로, dd = 'ㅇㅇ'을 영문 자판으로 친 것)

클로드가 "진행할까요?", "올릴까요?" 등으로 물었을 때 사용자가 위 답 중 하나를 치면
**재확인 없이 바로 그 작업을 진행**한다. 다른 맥락에서도 동일하게 긍정으로 해석한다.

## 저장소 규칙

- `accounts.json`(게임 계정 정보)과 `local_config.json`(머신별 설정)은 **절대 GitHub에 올리지 않는다** (gitignore 됨).
- `daya_counts.json`·`daya_history.json`·`daya_regions.json`(다야 측정값과 OCR 좌표/캡처영역)은 **머신별 데이터** — 실제 저장 위치는 `%LOCALAPPDATA%\MoonAI` 이며 GitHub에 올리지 않고, 업데이트/git pull이 절대 덮어쓰지 않는다. 마지막 측정값과 등록 좌표는 사용자가 직접 바꾸기 전까지 그대로 유지한다.
- 좌표(coords.json, island_coords.json)와 오림의일기장 이미지(reroll_templates/)는 공유 대상.
- 코드 수정 후에는 항상 바탕화면으로 복사(배포)하고 워치독(`LineageM_Watchdog` 예약 작업)으로 메인런처를 재시작한다.

## 메인런처는 절대 꺼진 채로 두지 않는다

- `git pull`·업데이트·좌표 편집 등으로 메인런처를 종료했으면, 작업이 끝난 뒤 **항상 다시 실행**한다.
  중간에 오류가 나더라도 **런처부터 먼저 다시 띄우고** 나서 오류를 보고한다.
- 종료가 필요한 작업(coords.json 편집 등)은 **종료 → 작업 → 재시작을 한 번에** 끝내고,
  사용자에게 "꺼져 있는 상태"를 남기지 않는다.
- 메인런처 [🔄 업데이트] 버튼도 동일 — 파일 복사가 실패해도 예외를 잡아 재시작 단계로 넘어가고,
  워치독 → 직접 실행 순으로 폴백하며 실제 기동까지 확인한다.
- **재시작한 런처는 최소화하지 말고 창을 띄워서 보여준다** — 사용자가 요청한 재시작/업데이트 후에는
  런처 창을 화면에 복원(SW_RESTORE)해 보여줄 것. 이후 정리는 10분 유휴 자동 최소화가 담당하고,
  불필요하면 사용자가 직접 최소화한다. (단, 새벽 워치독 자동 시작(04:00/04:50)은 최소화 유지)

## 좌표 자동 백업·복구 (모든 컴퓨터 공통)

- 좌표 파일(coords.json·island_coords.json·local_config.json)은 **런처 시작 시 하루 1회** +
  **업데이트 실행 시마다** `%LOCALAPPDATA%\MoonAI\backups` 에 자동 백업된다 (약 한 달치 보관).
- 사용자가 "**좌표가 날아갔다/초기화됐다**"고 하면: 이 백업 폴더에서 **가장 최근의 정상 백업**
  (파일 크기·등록 좌표 개수로 판단)을 찾아 런처 실행 폴더의 coords.json 등으로 복원하고
  런처를 재시작해줄 것. git stash 목록(`git stash list`)에 '업데이트 자동백업'이 있으면 그것도 확인.
