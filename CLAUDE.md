# Moon-AI 프로젝트 — 클로드 공통 지침

## 지침 관리 규칙 (클로드 스스로 지킬 것)

- 사용자가 클로드의 행동 방식에 대한 **새 지침**을 내리면, 이 파일(CLAUDE.md)에 추가하고
  **자동으로 GitHub에 push** 한다 (따로 물어보지 않아도 됨).
- `git pull`을 받으면 이 파일의 최신 지침을 읽고 **그대로 따른다** — 어느 컴퓨터에서든 동일하게.

## 사용자 답변 축약어 (모든 컴퓨터 공통)

**응 = ㅇㅇ = dmd = dd** → 전부 "응(yes)" 이라는 뜻이다.
(dmd = '응'을 영문 자판으로, dd = 'ㅇㅇ'을 영문 자판으로 친 것)

클로드가 "진행할까요?", "올릴까요?" 등으로 물었을 때 사용자가 위 답 중 하나를 치면
**재확인 없이 바로 그 작업을 진행**한다. 다른 맥락에서도 동일하게 긍정으로 해석한다.

## 저장소 규칙

- `accounts.json`(게임 계정 정보)과 `local_config.json`(머신별 설정)은 **절대 GitHub에 올리지 않는다** (gitignore 됨).
- `daya_counts.json`·`daya_history.json`·`daya_regions.json`(다야 측정값과 OCR 좌표/캡처영역)은 **머신별 데이터** — 실제 저장 위치는 `%LOCALAPPDATA%\MoonAI` 이며 GitHub에 올리지 않고, 업데이트/git pull이 절대 덮어쓰지 않는다. 마지막 측정값과 등록 좌표는 사용자가 직접 바꾸기 전까지 그대로 유지한다.
- 좌표(coords.json, island_coords.json)와 오림의일기장 이미지(reroll_templates/)는 공유 대상.
- 코드 수정 후에는 항상 바탕화면으로 복사(배포)하고 워치독(`LineageM_Watchdog` 예약 작업)으로 메인런처를 재시작한다.

## 메인런처는 절대 꺼진 채로 두지 않는다

- `git pull`·업데이트·좌표 편집 등으로 메인런처를 종료했으면, 작업이 끝난 뒤 **항상 다시 실행**한다.
  중간에 오류가 나더라도 **런처부터 먼저 다시 띄우고** 나서 오류를 보고한다.
- 종료가 필요한 작업(coords.json 편집 등)은 **종료 → 작업 → 재시작을 한 번에** 끝내고,
  사용자에게 "꺼져 있는 상태"를 남기지 않는다.
- 메인런처 [🔄 업데이트] 버튼도 동일 — 파일 복사가 실패해도 예외를 잡아 재시작 단계로 넘어가고,
  워치독 → 직접 실행 순으로 폴백하며 실제 기동까지 확인한다.
